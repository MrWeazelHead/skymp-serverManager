📖 SkyMP Documentation

The definitive guide to your multiplayer server

Chapters

📜 Chapter I: Setup & Architecture

Skylow Multiplayer (SkyMP) is a complex system that bridges multiple Skyrim instances together. To ensure a stable server, your foundation must be solid.

⚔ The Architecture

  • The Client: A mod that runs inside your Skyrim SE. It handles animations, movement, and menus.
  • The Server: A standalone program that synchronizes the world state (chests, NPCs, players) for everyone.
  • The Panel: The interface you are using right now to manage the server process.

⚙️ Essential Requirements

Before you begin, ensure both you and your players have:

  • Skyrim Special Edition: Version 1.5.97 is the most tested.
  • SKSE64: Required for the client to function.
  • Visual Studio 2022: Required if you intend to build from source code.
💡
Tip: Always run your Skyrim via skse64_loader.exe to ensure mods load correctly.

⚒ Chapter II: Building & Compiling

Building is the process of turning source code into the actual programs you run. In SkyMP, this happens via CMake and Visual Studio.

🏰 Building the Solution

  1. Open build/skymp.sln in Visual Studio 2022.
  2. Set the configuration to Release in the top toolbar.
  3. Go to Build -> Build Solution.

📁 Where do the files go?

Once the build processing is finished, your ready-to-use programs appear in build/dist/:

  • dist/server: Contains the server scripts and executable.
  • dist/client: Contains the files you need to copy into your Skyrim folder to play.
⚠️
Important: If you change C++ code or update the repository, you MUST rebuild to apply changes!

⚔ Chapter III: Addons & Plugins

Addons extend your server's logic. There are two ways to add them, depending on your workflow.

📦 Manual Placement (Post-Build)

Use this if you just want to add a mod or script to your currently running server.

  • Mods (.esp): Place in dist/server/data/ and add to the Load Order in Settings.
  • Scripts (.pex): Place in dist/server/data/scripts/.
📜
These changes are immediate but will be overwritten if you delete your dist folder!

🏗 Source Placement (Pre-Build)

Use this for permanent additions that you want to be part of every build you make.

  • Server Scripts: Add your source files to skymp5-scripts/.
  • Client Addons: Place files in the appropriate skymp5-client subdirectories before building.
⚒
Content here requires a **Rebuild** in Visual Studio to take effect.

🔄 Chapter IV: Updating the Server

Keeping your server updated ensures you have the latest bug fixes and features.

📦 Manual Update (Simple)

  1. Download the latest source code ZIP from GitHub.
  2. Extract it and replace your old files (keep your dist/server/world folder if you want to save progress!).
  3. Run the Build process again in Visual Studio.

🌿 The Git Path (Recommended)

Use this if you use Git tools (like GitHub Desktop or Command Line).

git pull origin main git submodule update --init --recursive

After pulling, open Visual Studio and rebuild the solution.

🛡 Chapter V: Server Maintenance

A healthy server requires regular upkeep.

🧹 World Resets

If you want to clear all player changes (opened doors, looted chests, dropped items), navigate to the Files tab and use the Reset World button.

🛡 Backups

Always create a manual backup before updating your server or changing major settings. Backups are stored in panel-data/backups.

📊 Performance History

Check the Sessions tab to see how long your server has been running and identify any patterns in crashes.

🌐 Chapter VI: Hosting & Deployment

Moving from a local test to a living, breathing online world requires proper networking.

🏠 Hosting Strategy

  • Local PC: Great for testing alone or with close friends. Requires your PC to stay on.
  • VPS (Virtual Private Server): Best for 24/7 Servers. We recommend Ubuntu 22.04 for Linux hosting or a Windows Server instance.

🔓 Networking & Ports

For players to find you, you MUST open these ports in your router or VPS firewall:

  • 7777 (UDP): The Game Port. Essential for synchronization.
  • 3847 (TCP): The Panel Port. Required to access this interface remotely.

⚙️ Final Settings

Go to the Settings tab and ensure:

  1. Server Name: Set something epic that players will see.
  2. IP Address: Use 0.0.0.0 to listen on all available connections.
  3. Public IP: Ensure your players have your actual Public IP to connect.

🎮 Chapter VII: Playtesting & Connection

Now that the server is running, it's time to connect.

📦 Client Setup

  1. Inside your build, go to dist/client/Data.
  2. Copy all contents inside into your Skyrim Special Edition/Data directory.
  3. Ensure you have Skyrim Platform installed from Nexus Mods.

🔗 In-Game Connection

  1. Launch Skyrim via skse64_loader.exe. The SkyMP client mod will load automatically.
  2. Once at the main menu, press the [F2] key on your keyboard to open the Server Browser UI.
  3. Mouse captured? If you cannot move your mouse to click the UI, press [F6] to toggle mouse focus free from the game.
  4. Find your server in the Master Server List or click Direct Connect and enter your server's IP address (use 127.0.0.1 if hosting locally).
💡
Persistent IP: SkyMP saves the last IP you correctly connected to internally!

👑 Server Administrator Powers

As the Server Administrator, you have control over the server via the Console tab:

say "Greetings, players!" kick "PlayerName" ban "PlayerName"

🤖 Chapter VIII: Tips, Tricks & Automation

Once you are familiar with the standard playtesting loop, you can automate your workflow to join your server faster.

🚀 1. The Auto-Connect Fast Load

Because the SkyMP client saves your last joined Server IP to its internal storage, you do not need to use the Server Browser every time. Instead, you can bypass the main menu completely to load instantly onto the server!

  • When the main menu appears, press ~ to open the developer console.
  • Type coc riverwood and press Enter.
  • You will instantly load into Riverwood and automatically reconnect to your server!

💻 2. The Auto-Launch Batch Script

Create a simple Play.bat file inside your Skyrim directory (next to skse64_loader.exe) to easily launch the game without opening Steam or Mod Organizer manually (useful for rapid development runs):

@echo off echo Launching Skyrim Multiplayer... start "" "skse64_loader.exe" -waitfortesting

🎭 3. Logging & Debugging in Skyrim

If you are writing custom Papyrus or JavaScript plugins, you can watch your debug prints live by opening the `~` console inside Skyrim. You will see messages like [Script] Hello World appear.

⚡ Quick Start Guide

🚀

A quick guide for a running server.

  1. Build: Build the solution in Visual Studio 2022 (Release).
  2. Client Setup: Copy dist/client files to your Skyrim Data folder.
  3. Run Server: Run Start Panel.bat and click Start Server.
  4. Play: Launch Skyrim via SKSE, press [F2], and join your server.

📝 Checklist

  • [ ] Skyrim version 1.5.97?
  • [ ] Port 7777 (UDP) Open?
  • [ ] SKSE64 installed?
  • [ ] Skyrim Platform (Nexus) installed?
